# Project Rules

## 语言规则

- 与用户交流统一使用中文。
- 文件名、目录名、变量名、函数名、类名、接口名统一使用英文。
- 代码注释统一使用中文。
- 面向初学者解释时，先说明这一步做什么、为什么做、可能影响哪里。

## 代码与文档风格

- 本仓库主要由 Markdown 规范、方法论文档和模板组成。
- 修改时优先保持现有写法，不做无关重构。
- 每次只更新与当前目标直接相关的文档，避免制造文档噪音。
- 新增模板应放在 `templates/` 目录。
- 新增阶段说明应放在 `methodology/` 目录，除非只是补充现有阶段。

## 目录与命名规范

- `SKILL.md` 是主执行规范。
- `methodology/` 存放各阶段详细方法论。
- `extractors/` 存放候选提取器 prompt。
- `templates/` 存放产出模板。
- 蒸馏产物默认放在 `books/<book-slug>/`。

## 技术栈约束

- 当前项目是纯 Markdown / Prompt 工程项目。
- 不引入构建工具、数据库、后端服务、外部 API 或付费云服务。
- 第一版优化只调整规范和模板。

## 已确认不可随意变更的规则

- 不改变 cangjie-skill 的核心定位：把长内容蒸馏成可执行 agent skills。
- 不改变 RIA-TV++ 的主流程顺序。
- 不删除原有三重验证、压力测试、用户确认、审计轨迹等核心机制。
- 新增测试类型必须尽量保持 `test-prompts.json` 对 darwin-skill 的兼容性。

## 安全与配置要求

- 不提交密钥、令牌、账号信息或本地私有路径。
- 不新增需要联网运行的强制步骤。
- 上传 GitHub 前先检查 Git diff。

## 开发边界与非目标

- 本轮不实现自动化程序或图形界面。
- 本轮不做跨书全局索引、usage-log 反馈闭环、manifest.json 机器可读升级。
- 本轮不改 README 的中英日完整国际化同步，只在必要时更新中文主说明。
