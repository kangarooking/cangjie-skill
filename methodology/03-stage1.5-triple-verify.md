# 阶段 1.5 — 三重验证筛选

## 目标

从候选单元池里,筛出**真正值得做成独立 skill 的方法论单元**。通不过的降级为 example / 引用 / 术语,但不独立成 skill。

这是 cangjie-skill 区别于"书摘工具"的核心质量门。本阶段新增两个增强机制:

1. **双评审 + 仲裁**: 降低单个 agent 主观判断带来的漂移。
2. **V1 自适应**: 让短视频、播客、访谈等短内容不被长书标准误杀。

## 内容体量分级

阶段 1.5 开始前,先确认 `SKILL.md` 输入要求中记录的内容体量:

| 分级 | 参考标准 | V1 证据要求 |
|---|---|---|
| 长书 | 10 万字以上,或多章节系统性著作 | 至少 2 个内容内独立语境 |
| 中篇 | 2–10 万字,或较完整课程/长文 | 至少 2 个内容内独立语境 |
| 短内容 | 少于 2 万字,或少于 40 分钟转写稿 | 1 处内容内佐证 + 1 处外部可佐证 |

短内容的外部佐证不是让 agent 随意扩写,而是让它用已有知识判断这个主张是否能被独立经验、公开常识或相关领域知识支撑。外部佐证必须写明来源类型,例如"行业通行实践"、"经典理论相符"、"作者其他公开表达一致"。

## 三重验证 (全部通过才录取)

### V1 — 跨域验证 (Cross-domain)

**问题**: 这个单元是否有足够独立证据说明它不是孤立表达?

- 长书 / 中篇: 在书中**至少 2 个独立的语境**下有佐证。
- 短内容: 至少 1 处内容内佐证,再加 1 处外部可佐证。
- "独立"的含义: 不是同一个案例换个说法,而是两个不同的故事/不同的章节/不同的对象都在讲同一个道理。
- **通过**: 《穷查理宝典》中"逆向思维"在投资决策、避免灾难、教学方法三个独立场景中都出现。
- **不通过**: 某个漂亮句子只在一章里出现过一次,也没有外部可佐证的稳定方法论价值。

**证据等级**:

- **A**: 内容内多处独立佐证。
- **B**: 内容内单处佐证 + 外部可佐证。仅短内容可用。
- **C**: 证据不足,不得进入阶段 2。

**为什么**: 在多个语境中反复出现的,才是作者真正想传达的稳定方法论,而不是一时兴起的表达。短内容本身没有足够篇幅反复展开,所以需要用证据等级保持诚实。

### V2 — 预测力测试 (Predictive Power)

**问题**: 能用这个单元,推导出书里没明说的某个问题的答案吗?

- 自己设计一个书中没直接讨论过的场景。
- 尝试用这个方法论去分析它。
- **通过**: 能得出一个有意义、非平庸的结论。
- **不通过**: 只能得出"努力就会成功"之类的废话。

**为什么**: 真正的方法论必须有**外推能力**。如果它只能复述书里的例子,它是描述不是方法。

### V3 — 独特性检验 (Exclusivity)

**问题**: 这个单元是否是"任何聪明人都会说的常识"?

- 如果把作者名字抹掉,一个对书本领域毫无了解的聪明人也能说出来,不通过。
- 必须是作者**独特视角 / 反直觉见解 / 独特术语体系**。
- **通过**: 段永平的"stop doing list" — 主动列出不做什么,反常识。
- **不通过**: "要尊重时间" — 太常识了,没人需要一个 skill 来告诉自己这个。

**为什么**: 常识不需要 skill 承载,Claude 本身就知道。只有作者的**差异化见解**才值得固化成 skill。

## 双评审 + 仲裁流程

1. 把阶段 1 的 5 个 `candidates/*.md` 合并成一个总候选池。
2. 去重: 同一方法论被多个 extractor 提取的,合并成一条。
3. 启动两个独立评审 agent,分别对每条候选执行 V1 / V2 / V3。
4. 两个评审都通过: 进入 `verified.md`。
5. 两个评审都淘汰: 进入 `rejected/<id>.md`。
6. 两个评审结论不一致: 进入第三个仲裁 agent;如果仲裁仍不清晰,升级为用户确认项。
7. 在 `verified.md` 和 `rejected/` 中记录评审一致性,便于后续复盘。
8. **用户轻确认** ★: 把"通过的 N 个候选标题 + 淘汰的 M 个 + 分歧仲裁项 K 个"展示给用户,问一句"这 N 个会做成 skill,有想捞回或砍掉的吗?" 得到确认再进入阶段 2。

如果当前环境没有 sub-agent 能力,可以由主流程串行模拟两个评审,但必须在 `verified.md` 中标明 `review_mode: fallback_single_agent`,可信度低于独立评审。

## 输出模板 (verified.md 单条)

```yaml
id: f01
title: 逆向思维
type: framework
content_scale: long_book
evidence_grade: A
review_mode: dual_agent
review_consensus: agreed_pass
V1_cross_domain:
  passed: true
  evidence:
    - 第 3 讲: 投资决策场景
    - 第 7 讲: 工程设计场景
    - 第 11 讲: 教学方法场景
V2_predictive_power:
  passed: true
  novel_question: "如果面试官问我一个不知道答案的问题该怎么办?"
  derived_answer: "逆问'我最不希望他认为我是什么样的人',从这个反面倒推应该展现什么"
V3_exclusivity:
  passed: true
  why_not_common: "常识是'要多想',逆向思维是'优先反着想' — 这是反直觉的排序"
arbitration:
  needed: false
→ 进入阶段 2
```

## 常见失败模式

1. **V1 作弊** — 把同一例子换个说法算两处。要求: 必须是不同章节 + 不同对象 + 不同结论。
2. **短内容滥用外部佐证** — 外部佐证只能补足短内容篇幅限制,不能把内容里没有的方法论硬塞进去。
3. **V2 作弊** — 用一个其实书里讨论过的类似问题冒充"新问题"。要求: 新问题应该让人第一眼不知道书里怎么说。
4. **V3 过松** — 只要"说得比较文雅"就认为不是常识。要求: 看**内容**本身是否反直觉,而不是措辞。
5. **仲裁走形式** — 仲裁必须解释为什么采纳一方,不能只写"同意评审 A"。

## 数量预期

经验上,一本方法论密集的书 (如《穷查理宝典》) 通过率约 30–50%。一本散文类书可能只有 5–10%。通过率过低 (<5%) 或过高 (>80%) 都要警惕:

- 过低: extractor 可能质量差,要重跑
- 过高: 验证标准可能太松
- 双评审一致率过低: V1/V2/V3 标准可能解释不清,需要先校准再继续
