# 阶段 -1 — 书籍适配判断

## 目标

在正式拆书前,先判断这本书是否适合被做成可调用的 AI skills。不是所有书都应该被硬拆成 skill pack。

产出: `books/<slug>/BOOK_FIT.md` (按 `templates/BOOK_FIT.md.template` 填充)。

## 为什么要先判断

book2skill 的目标是提取可迁移、可执行、可测试的方法论。如果一本书主要提供的是人物经验、审美体验、故事世界、史料汇编或文学表达,硬拆成 skill 会产生两类坏结果:

- 把故事、金句、观点误包装成方法论
- 为了凑数量生成 trigger 很宽、边界很虚的低质量 skills

阶段 -1 的任务不是评价一本书"好不好",而是判断它适合输出什么产物。

## 四类结论

| 结论 | 含义 | 下一步 |
|---|---|---|
| `full_skill_pack` | 方法论密度高,适合完整 skill 化 | 继续阶段 0,预估 8-25 个 skills |
| `partial_skill_pack` | 有少量可执行方法,但大部分内容不适合 skill 化 | 继续阶段 0,预估 3-8 个 skills,严格控制数量 |
| `alternate_artifact` | 价值很高,但更适合其他结构化产物 | 向用户建议人物库、世界观库、案例库、术语库、论证地图等 |
| `not_suitable` | 当前材料不足或目标与 skill 化不匹配 | 停止,向用户说明缺什么或为什么不建议 |

## 评分维度

每项 0-5 分,总分 25 分。分数用于辅助判断,不能替代理由。

| 维度 | 高分信号 | 低分信号 |
|---|---|---|
| 方法论密度 | 有框架、流程、原则、清单、决策规则 | 主要是叙事、抒情、史料、观点散点 |
| 可执行性 | 能转成步骤、判停、输入输出 | 只能"体会"或"欣赏",难以执行 |
| 迁移性 | 可跨场景复用 | 只在书中单一情境成立 |
| 证据/案例支撑 | 作者多处用案例或反例支撑 | 只有单次表达或漂亮句子 |
| 边界清晰度 | 书中有反例、适用范围或失败模式 | 方法泛化空间太大,容易乱用 |

建议阈值:

- 20-25: `full_skill_pack`
- 14-19: `partial_skill_pack`
- 8-13: `alternate_artifact`
- 0-7: `not_suitable`

## 按书类快速判断

| 书类 | 默认判断 | 注意 |
|---|---|---|
| 方法论书、管理书、投资书、运营书、写作/设计手册 | 通常 full 或 partial | 仍要防止把常识硬做成 skill |
| 访谈、书信、演讲集 | 通常 partial | 适合抽原则和判断框架,不适合每段都拆 |
| 小说、诗歌、散文 | 通常 alternate_artifact | 更适合人物、世界观、主题、风格或情节结构库 |
| 纯史料、编年史、资料汇编 | 通常 alternate_artifact | 更适合时间线、案例库、事件索引、证据地图 |
| 哲学/思想经典 | partial 或 alternate | 只有能形成可执行判断程序时才做 skill |
| 医学、法律、金融高风险材料 | partial 或 not_suitable | 必须加安全边界,不得替代专业建议 |

## 执行步骤

1. **读取最小判断材料**
   - 书名、作者、出版年
   - 目录、前言/导论、结语
   - 1-2 个代表章节
   - 用户想要的用途

2. **打分并写理由**
   - 按 5 个维度分别打 0-5 分
   - 每项必须写一句证据:来自目录、章节结构、作者声明或文本样本

3. **选择产物形态**
   - 如果 full / partial: 说明预估 skill 数量和优先主题
   - 如果 alternate: 选择最合适的替代产物
   - 如果 not_suitable: 说明缺少什么材料或目标为何不匹配

4. **用户检查点**
   - 展示 `BOOK_FIT.md`
   - 若不是 full / partial,必须问用户是否接受替代产物或仍要强行拆 skill

## 替代产物建议

| 不适配原因 | 更适合的产物 |
|---|---|
| 小说/戏剧/史诗 | 人物关系图、世界观设定库、主题地图、情节结构库 |
| 散文/诗歌 | 风格分析、意象词典、主题索引、写作风格 skill |
| 纯史料 | 时间线、事件索引、案例库、证据地图 |
| 传记 | 人物决策模式、关键事件案例库、人生阶段地图 |
| 哲学文本 | 概念词典、论证地图、问题谱系、少量判断 skill |

## 质量门

- [ ] 已产出 `BOOK_FIT.md`
- [ ] 5 个维度都有分数和证据
- [ ] 结论属于四类之一
- [ ] 若结论不是 full / partial,已给替代产物建议
- [ ] 已向用户展示并得到继续方向确认

## 常见失败模式

1. **把书的价值等同于 skill 适配度**: 一本伟大的小说也未必适合做 skill pack。
2. **为了数量硬拆**: partial fit 的书不要硬凑 10 个以上 skills。
3. **忽略用户用途**: 同一本书用于写作模仿、人物研究、决策训练时,适配结论可能不同。
4. **跳过替代产物**: 判断不适合 skill 化后,仍应告诉用户更好的结构化方式。
